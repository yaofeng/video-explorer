创建一个视频库浏览器，无需视频预览和播放，只需要像文件浏览器一样对视频文件进行查看。
使用 python 作为后端，vue3 作为前端，使用 tailwandcss 实现页面样式。
子目录如下：
backend - 后端代码
frontend - 前端代码
bin - 启动脚步，包含 start-dev.sh / start.sh / stop.sh / restart.sh / build.sh
docker - docker 相关文件，包含 Dockerfile / docker-compose.yaml 等
.env - 环境变量

脚本功能如下：
start-dev.sh - 用 nohup 使用开发模式分别启动前后端项目，并生成 app.pid 文件（包含前后端）
start.sh - 用 nohup 使用生产模式启动后端项目，并生成 app.pid 文件
stop.sh - 根据 .pid 文件停止相关进程
restart.sh - 停止当前进程并使用生产模式启动后端项目
build.sh - 构建前端项目并放到 backend 的静态资源目录

.env 环境变量说明：
DATA_PATH - 数据目录，包含 config.yaml文件、logs目录、cache目录等，默认为当前目录
IP_WHITE_LIST - IP 白名单，只有白名单内的 IP 地址可以访问本服务

本项目有如下功能：
1、视频文件浏览功能
卡片式布局展示目录中的视频文件。
页面最上方是视频一级菜单，对应的一级目录，左侧为二级菜单，对应视频的二级目录，中心区域按照三级（含以上级别，需要展开）目录进行分组，每一组中，用卡片的样式展示视频文件。
视频卡片展示的内容包含缩率图、文件名称、编码方式（H264/HEVC等）、文件大小（单位G）、时长、分辨率（1080P/720P等），缩率图在上方，文件名在缩率图的下方（两行展示），编码方式在缩率图左上角，分辨率在右上角，时长在左下角，大小在右下角。
首次打开目录时会读取视频相关信息，生成视频的描述信息（二进制文件，包含缩率图，编码方式，时长，分辨率等），保存到 cache 目录从，cache 目录结构和文件名与视频目录和原文件名保持一致。
后续打开目录时会补充新增视频的描述文件，同时删除原视频不存在的描述文件，并且判断文件修改时间晚于描述文件修改时间的，进行更新。
视频缩率图优先使用视频文件的封面，如果文件不包含封面，则通过抽帧的方式获取，取开头 3:30 的帧。
缩率图的展示使用 16:9，如果获取的缩率图不是该比例，则等比例缩小，不要拉伸或截取。

2、设置功能
所有的配置都保存到 DATA_PATH 的 config.yaml 文件中。
video_path_list - 视频目录列表，视频文件浏览功能需要选择该列表的一个目录，然后浏览其中包含的视频
page_size - 视频文件浏览功能每页的视频数，默认为 0 ，表示部分页全部展示
column_size - 视频文件浏览功能中每行的视频数量
